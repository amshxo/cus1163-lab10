# Lab 10 Analysis Questions
## 1. Security Risk A world-writable directory is way more dangerous than just a single world-writable file. If a directory is 777, any random user can delete files they don't own, move things around, or even upload their own malicious scripts. With a file, the damage is usually stuck to just that one file, but a writable directory gives an attacker a foothold to mess with the entire folder structure. 
# 2. Permission Bits The -perm -002 flag is looking specifically for the "world-writable" bit, meaning anyone on the system can modify it. On the other hand, -perm /111 is checking if any of the execute bits (owner, group, or others) are set. We use both because we're looking for two different types of risks: one where people can change our data, and one where files are marked as executable when they shouldn't be (like a .txt or .conf file). ## 3. Real-World Impact If I saw 50 world-writable files on a live server, I'd immediately flag it to the security lead. I would then use a script or the chmod command to strip those permissions back to something safe like 644 or 600. I'd also check the logs to see if those files were actually touched or modified recently by anyone suspicious. ## 4. Process Substitution When you use a pipe like find ... | while read, it actually starts a subshell. This means any variables you change inside the loop (like our count variable) get lost the second the loop ends. By using while read ... done < <(find ...), the loop runs in the current shell, so our counter actually keeps its value and we get the right total at the end. ## 5.
